Important Concepts

Lesson 1: Introduction

What is coding

Coding is the process of telling a computer what to do, in a language it understands.

Thinking Like a Programmer

There are five processes that programmers use to tackle problems

  1. Procedural Thinking: is the process of taking problems and breaking it down into a series of steps to address the problem.
  2. Abstract Thinking: is the process of using logic to get to the core of an abstract problem, e.g. one apple plus one apple equals two apples. The core logic behind it is 1+1 = 2, even though you are using apples to replace the numbers.
  3. Systematic Debugging: the process of identifying problems and tracking it down to its cause to fix the problem
  4. Technological Empathy: the process of understanding how the computer sees things
  5. Systems thinking: Thinking like a programmer and using the above steps to identify and fix the problem.
What is the Internet

The internet is a collection of data. Namely, HTML documents that are contained on servers and accessed by myself or others using a web brower. HTML documents are comprised of text context (what I see), markup (what it looks like), references (other documents like images and videos) and links (links to other pages).

Lesson 2:HTML and CSS

What is HTML

HTML stands for Hyper Text Markup Langauge. This is the standard programming language used to make web pages.

HTML is the main code that makes up webpages, and the internet as a whole. Additionally when coding, HTML is the blueprint like that of a house. HTML defines what and where everything is on the page. Just like how a blueprint defines what and where rooms and things are of a house.

What is CSS

CSS stands for Cascading Style Sheet.CSS on the other hand is a style, which describes how a page will look, very simliar to choosing what color walls to put in your house.

What is DOM

Document Object Model - a cross platform and language independent convention for representing and interacting with objects in HTML (and other markup languages). The nodes of every document are organized in a tree structure, called the DOM tree.

Structure

Like all things, HTML has a specific structure when being used. It's organized into a "tree-like" structure that allows for easy organization while also allowing rules to be applied to every branch or specific branches when code is written.

Thinking in Boxes

Along with organizational structure, everything on a webpage is defined or put into a box. This is a form of organization to allow you to help keep your webpage/HTML document organized and neat looking. It also allows you to "jazz" it up by modifying the shape and what you want to put into your page, i.e. pictures, links, etc. Like moving into or out of a house, this keeps everything in a group and defines where it will go.

Lesson 3: Using HTML and CSS

The Coding Process

As stated earlier, coding is the process of telling a computer what to do in its own language. To do this we use a specific set of rules paired with our organizational structure to create a document that the computer then translates into a webpage.

Tags and Elements

We use tags to define elements (or what makes up HTML code). For example a tag such as <b> will define that you want the text to contain an element that is "bold". Examples of tags are: <b> - bold, and <em> - italics.

Inline and Blocks

Inline objects are tags you use in line of code to tell the computer how you want that code to display, e.g. using <b> to show that you wanted text in bold. Blocks are tags that you use to put just that, blocks around your text to create form and spacing. You can also modify these blocks by defining its height and width.

Attributes

Attributes are properties of HTML elements. For example, we can use the <href> tag to create an anchor which is basically a link to another document (page, image, etc.). An example of an <href> tag is this: <a href="www.yahoo.com"> Yahoo! </a>.

Using CSS

CSS is used to define the styles used in your webpage. These styles define what color you want certain text to be, or what color you want the background, etc.

CSS is similar to your regular HTML page, as you use a certain definitions (tags) to define these. Organization is key as you define where and what you want to have in CSS. For example, we would organize text as a class "text", then we could define say: color: red, or background-color: gray, etc.

Syntax

Syntax is like using proper grammar in coding. If things are not written in a specific way the computer will not know how to translate it and something could go wrong. For example typing: <b><em> syntax </em></b> would be incorrect. To be proper syntax <em> must be closed before <b> because that is how we defined the open tags.

Repetition is Painful

Coding is a time consuming process, and it becomees more time consuming when you have to define everything on your page from text to style. So by organizing things into using the div tag, we can group things into say; a "title", "text", "description", etc. So that we can easily style that certain group in CSS.

More importantly it is a huge pain to have to go back and fix every little thing if it is not organized. When things are grouped it also becomes easier to figure out what went wrong so that this process becomes somewhat easier. In the long run by doing this we don't have to have as much repetition and it makes us easier to write code and fix it.
